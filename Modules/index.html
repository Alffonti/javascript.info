ES Modules

Import and Export statements in JavaScript, only function within ES modules. To convert a script into an ES module, simply add the type="module" attribute to the script tag.

However, it's important to note that ES modules are subject to CORS (Cross-Origin Resource Sharing) restrictions. When using the file protocol (file://) to access a module directly from the local file system, browsers block the access. This is because, in such cases, no server is involved, and CORS headers are not sent. The absence of appropriate CORS headers leads to the browser rejecting the request, thus preventing ES modules from being loaded and executed correctly.

To successfully use ES modules in the browser, it is recommended to either run a local development server, such as Node.js static-server, VSCode Live Server extension or use a web server to serve your web pages and scripts using the HTTP protocol. By doing so, the server will send the appropriate CORS headers, allowing the browser to grant access to the module.
